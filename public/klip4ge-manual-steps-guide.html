Replace onboarding MP4 placeholders with real screen recordings

6 videos · Each ~60–90 seconds · 1080p H.264 MP4

⏱ ~30 min
ℹ️
Why this matters: The onboarding video player auto-plays a short walkthrough the first time a user visits each page. Right now it silently fails because the MP4 files are 40-byte stubs. Replacing them makes the onboarding tour actually play.

1

Record each screen

Open klip4ge.app (or your staging URL) in a browser. Use one of these free tools to record your screen:

ToolPlatformHow to open
Loom (recommended)Browser extension / Mac / Winloom.com → "New Recording"
QuickTime PlayerMac onlyFile → New Screen Recording
Xbox Game BarWindows 10/11Win + G → "Capture"
OBS StudioMac / Win / Linux (free)obsproject.com

Record at 1920×1080. Show your cursor. Narrate what you're doing (or add a voiceover later — the player has a mute button so silent works too).

2

Export as H.264 MP4

Save each recording as .mp4 with H.264 video codec. Most tools default to this. In Loom: download → "Original quality". In OBS: Settings → Output → Recording Format = mp4. Target file size: under 20 MB each (compress with HandBrake if needed — RF 23, web-optimised preset).

3

Create a poster thumbnail (JPEG) for each video

Take a screenshot of the first frame (or a good mid-point). Save as .jpg at 1280×720. Name it to match the video: dashboard-poster.jpg, saves-poster.jpg, etc.

Quick method — open the MP4 in VLC → pause at 0:01 → Video → "Take Snapshot".

3

Set the published Chrome Extension ID in Layout.jsx

1-line code edit · Locks the auth token relay to your specific extension

⏱ ~5 min
⚠️
Do this AFTER publishing the extension to the Chrome Web Store. Until then, the placeholder string "REPLACE_WITH_PUBLISHED_EXTENSION_ID" causes chrome.runtime.sendMessage to fail silently (which is safe — the app still works, the extension just won't receive the auth token automatically).
1

Publish the Klip4ge MV3 Extension to Chrome Web Store

If you haven't already, submit the extension at:

chrome.google.com/webstore/developer/dashboard

Required assets (all already in the repo under extension/):

  • extension/manifest.json — MV3 manifest
  • extension/icons/ — 16, 32, 48, 128 px PNGs
  • extension/popup/popup.html + popup.js
  • extension/background.js
  • extension/content.js
  • extension/store-promo-1280x800.png — promo image (replace placeholder with real screenshot)

Zip the extension folder: cd clipforge && zip -r klip4ge-extension.zip extension/ then upload on the CWS dashboard.

2

Find your Extension ID

After the extension is approved (or even while in draft/testing):

  1. Go to Chrome Web Store Developer Dashboard
  2. Click your Klip4ge extension item
  3. The URL will contain the ID: …/detail/abcdefghijklmnopabcdefghijklmnop
  4. Or: open Chrome → go to chrome://extensions → enable "Developer mode" (top-right toggle) → the ID appears under your extension name

The ID is always a 32-character lowercase string like: abcdefghijklmnopabcdefghijklmnop

3

Edit Layout.jsx — line 75

Open src/Layout.jsx in your editor. Find this block (around line 75):

// BEFORE
            "REPLACE_WITH_PUBLISHED_EXTENSION_ID", // TODO: set after Chrome Web Store submission

Replace it with your actual 32-character extension ID:

// AFTER (example ID — use YOUR real one)
            "abcdefghijklmnopabcdefghijklmnop", // Klip4ge MV3 Chrome Extension
🚨
Security note: The Extension ID locks which Chrome extension can receive the auth token. If you leave it as undefined in production, any extension on the klip4ge.app origin could theoretically receive the token. Always set a real ID before going live.
4

Commit and push

git add src/Layout.jsx
git commit -m "feat(extension): set published Chrome Web Store extension ID in auth relay"
git push origin main
✅
Verify: Install the published extension from CWS, sign into klip4ge.app, then click the extension icon — it should show you as already signed in without entering credentials again.
4

Replace the APK signing fingerprint in assetlinks.json

Required for Android TWA — lets Chrome trust your app as native

⏱ ~15 min
ℹ️
What is this? A Trusted Web Activity (TWA) is how your PWA gets listed on Google Play as a "native" Android app. The assetlinks.json file tells Android Chrome: "this website is owned by the Android app with this signing key." Without the correct fingerprint, the TWA will open in a browser tab instead of full-screen, and Play Store submission will fail verification.
⚠️
Prerequisites for this step: You need a Google Play Developer account ($25 one-time fee) and Java JDK installed on your computer (for keytool). If you're using Google Play App Signing (recommended), the fingerprint comes from the Play Console instead — see Sub-step 2b.
1

Build the TWA APK using PWABuilder (easiest method)

You don't need Android Studio. Use PWABuilder — it builds the TWA for you from your manifest URL:

  1. Go to pwabuilder.com
  2. Enter your URL: https://klip4ge.app → click "Start"
  3. PWABuilder scores your manifest and service worker — click "Package for stores"
  4. Select "Google Play" → click "Generate Package"
  5. Fill in the form:
    • Package ID: app.klip4ge.app
    • App version: 1
    • Version name: 1.0.0
    • Signing key: Choose "New" to let PWABuilder generate a keystore, or "Existing" to upload yours
  6. Download the .zip — it contains your .aab (Android App Bundle), a .keystore file, and a assetlinks.json with the correct fingerprint already filled in
2

Get the SHA-256 fingerprint from your keystore

Option A — from the PWABuilder-generated keystore:

PWABuilder includes the fingerprint in the downloaded assetlinks.json. Just copy it from there — skip to Sub-step 3.


Option B — from an existing keystore file using keytool:

Run this command in your terminal (replace the paths and alias with yours):

keytool -list -v \
  -keystore ~/path/to/klip4ge.keystore \
  -alias klip4ge \
  -storepass YOUR_KEYSTORE_PASSWORD

Look for the line that starts with "SHA256:". It looks like:

SHA256: AB:CD:12:34:EF:56:...:78:90:GH

Copy the full hex string without spaces or colons — you'll need it in the next step.


Option C — using Google Play App Signing (recommended for production):

  1. Upload your AAB to Google Play Console → your app → Release → Setup → App signing
  2. Under "App signing key certificate", copy the SHA-256 certificate fingerprint
  3. This is the one to use in assetlinks.json (Play re-signs your APK, so the original keystore fingerprint won't match)
3

Update assetlinks.json in the repo

Open public/.well-known/assetlinks.json. It currently looks like:

[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "app.klip4ge.app",
    "sha256_cert_fingerprints": [
      "REPLACE_WITH_YOUR_SIGNING_CERT_SHA256_FINGERPRINT"
    ]
  }
}]

Replace the placeholder with your real fingerprint (colons optional — both formats work):

[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "app.klip4ge.app",
    "sha256_cert_fingerprints": [
      "AB:CD:12:34:EF:56:78:90:AB:CD:12:34:EF:56:78:90:AB:CD:12:34:EF:56:78:90:AB:CD:12:34:EF:56:78:90"
    ]
  }
}]
💡
If you use both Play App Signing and a debug keystore, add both fingerprints as a JSON array: "sha256_cert_fingerprints": ["PRODUCTION_FP", "DEBUG_FP"]
4

Commit, push, and verify the hosted file

git add public/.well-known/assetlinks.json
git commit -m "feat(android): set APK signing certificate fingerprint in assetlinks.json"
git push origin main

After deploying, verify the file is publicly accessible:

curl https://klip4ge.app/.well-known/assetlinks.json

It must return the JSON with your fingerprint. Also check it with Google's tool:

Google Digital Asset Links validator ↗

✅
Verify on device: Install your TWA APK on an Android phone. Open it — the app should launch full-screen with no browser address bar (that's how you know the TWA trust link is working).
5

Rename the Base44 agent and update Assistant.jsx

Dashboard rename + 3-line code edit · Fixes the AI assistant chat

⏱ ~5 min
ℹ️
Why this matters: The Assistant.jsx page calls base44.agents.listConversations({ agent_name: "clipforge_assistant" }). If your Base44 dashboard has the agent registered under a different name, or if you want to rename it to match the Klip4ge brand, both the dashboard and the code must match exactly. Mismatched names return an empty conversation list and break the AI chat page.
1

Decision: rename or keep?

You have two choices:

OptionWhat to doBest if…
A Keep clipforge_assistant Do nothing — the code already uses this name. No change needed. The agent is already live and working in Base44
B Rename to klip4ge_assistant Rename in Base44 dashboard first, then update code You want full brand consistency OR the agent doesn't exist yet

⚠️ If users have existing conversations under the old name, renaming will lose that history. Only rename if you're pre-launch or willing to reset chat history.

2

Rename the agent in your Base44 dashboard

(Skip this sub-step if you chose Option A above)

  1. Log in at base44.com → your Klip4ge app
  2. In the left sidebar, click "Agents" (or "AI Agents" depending on your Base44 version)
  3. Find the agent named clipforge_assistant → click "Settings" or the ✏️ pencil icon
  4. Change the Name / Identifier field to: klip4ge_assistant
  5. Click Save
⚠️
Base44 agent names must be lowercase, no spaces, underscores only. The name is an internal API key — it does not display to end users.
3

Update Assistant.jsx — 3 lines

Open src/pages/Assistant.jsx. Find and replace all three occurrences of "clipforge_assistant" with "klip4ge_assistant":

// Lines 31, 37, and 82 — change all three

// BEFORE (3 places):
agent_name: "clipforge_assistant"

// AFTER (3 places):
agent_name: "klip4ge_assistant"

Quick command-line replacement (from repo root):

sed -i 's/"clipforge_assistant"/"klip4ge_assistant"/g' src/pages/Assistant.jsx

# Verify the change:
grep "agent_name" src/pages/Assistant.jsx

Expected output:

    const convs = await base44.agents.listConversations({ agent_name: "klip4ge_assistant" });
      agent_name: "klip4ge_assistant",
    const conv = activeConv || (await base44.agents.listConversations({ agent_name: "klip4ge_assistant" }))?.[0];
4

Run tests and push

# Run the full test suite to make sure nothing broke
NODE_OPTIONS="--max-old-space-size=1536" npx jest --forceExit

# Commit and push
git add src/pages/Assistant.jsx
git commit -m "feat(rebrand): rename Base44 agent from clipforge_assistant to klip4ge_assistant"
git push origin main
✅
Verify: Open klip4ge.app → navigate to the Assistant page → the AI chat should load your conversation history and respond normally. If the chat is blank/erroring, double-check the agent name in the Base44 dashboard matches exactly.
